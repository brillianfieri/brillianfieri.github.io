Store Register

The store register is a simple tool for sellers to manage their inventory and record sales transactions. It also includes a dashboard for gaining insights into their sales performance.

Summary

Key Features:
  • Registration and login using credentials.
  • Inventory and transaction management for admin role.
  • Simple dashboard to summarize sales performance for admin role.
  • Transaction input via cart checkout for admin and employee role.
  • Application of a dark mode scheme throughout the entire website.
  • Responsive design on various screen sizes.
Technologies Used:
  • Next.js 13 as the framework with the Pages directory.
  • PostgreSQL as the database system.
  • Prisma as the database connector.
  • Tailwind CSS framework for styling.
  • Next-auth library for authentication.
  • Code snippets sourced from Flowbites, Stack Overflow, and Tutorialspoint websites.

In-Depth

1. Account and Authentication

This app uses Next-auth to control the authentication process. There are two account roles available:

  • Employee

    In this application, the 'Employee' role is designed for regular employees, granting them the capability to utilize a shopping cart feature for item selection and subsequently proceed to checkout. Upon completing the checkout process, the transaction is recorded in the transaction history.

  • Admin

    This role holds higher privileges compared to the 'Employee' role. Admins possess all the abilities of the 'Employee' role, with the added capabilities to manage store inventory, delete past transactions, oversee user accounts, and monitor the change log across the entire application.

2. Dashboard

Dashboard can only be accessed by the Admins. This page displays charts for the total transactions in the current month, total sales by items, and total sales by category. Additionally, it features a chart comparing gross sales for the last three years.

Dashboard

3. Inventory Management

This page enables Admins to add, edit, and delete items in the inventory. Additionally, they can manage item categories by adding, modifying, or deleting category items. Employees in the 'Employee' role can access this page to view the inventory status but do not have the privilege to modify it.

4. Transaction

On the Transaction page, both Employees and Admins can create new transactions and access the history of past transactions. However, only Admins have the authority to delete past transactions.

Transaction (Admin Role)

Add Transaction (Employee Role): On this page, Employees and Admins can add items to the cart and proceed to checkout. The checkout process results in the creation of a new transaction history.

5. Admin Panel

This page contains two tabs: User and Log. On the 'User' tab, the admin can edit or delete users' accounts. For the 'Log' tab, the admin can review the activity logs across the app.
